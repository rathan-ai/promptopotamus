Introduction

A prompt is the input you provide to a Large Language Model (LLM) to get a specific output. Crafting an effective prompt involves model choice, wording, structure, and context—it’s a creative and iterative process.

Prompt engineering is the process of designing high-quality prompts that guide LLMs to produce accurate and relevant outputs.

Basic Prompting Techniques

Zero-Shot Prompting

The simplest prompt type: description only, no examples.

Classify the following movie review as POSITIVE, NEUTRAL, or NEGATIVE.

Advanced Prompting Techniques

Chain-of-Thought (CoT) Prompting

Encourage the model to think step-by-step for complex reasoning.

When I was 3 years old, my partner was 3 times my age. Now, I am 20 years old. How old is my partner? Let's think step by step.

Industry: Education

Simple: Create a Quiz

Create a 5-question multiple-choice quiz about the water cycle for a 5th-grade science class. Include an answer key.

Medium: Design a Lesson Plan

Act as a high school history teacher. Create a lesson plan for a 1-hour class on the main causes of World War I. The plan should include:
- Learning objectives
- Key terms: "Militarism", "Alliances", "Imperialism", "Nationalism"
- 15-minute lecture outline
- 20-minute group activity
- A simple assessment question to end the class.

Industry: Engineering

Simple: Explain a Technical Concept

Explain the concept of 'technical debt' to a non-technical project manager. Use a simple analogy related to home maintenance.

Medium: Review and Refactor Code

Act as a senior software engineer specializing in Python. Review the following function that calculates the Fibonacci sequence:

def fibonacci(n):
    if n <= 1:
        return n
    else:
        return fibonacci(n-1) + fibonacci(n-2)

Identify any potential bugs, suggest improvements for efficiency, and rewrite the code with better documentation and error handling for invalid inputs.

Risks & Caution

When crafting prompts, be aware of potential pitfalls and take precautions:

  • Ambiguity Risk: Vague prompts can lead to irrelevant or incorrect outputs. Always clarify the details.
  • Bias Caution: Prompts may inadvertently reinforce biases present in the model. Review and adjust language to be neutral.
  • Overfitting Concern: Too many examples can cause the model to follow patterns too rigidly. Strike a balance.
  • Privacy Risk: Never include sensitive personal data in prompts.
  • Misinterpretation: Models can misread instructions—test thoroughly before deploying.

Best Practices

  • Provide Examples: Use one-shot or few-shot examples to show the model the desired output.
  • Design with Simplicity: Prompts should be concise, clear, and easy to understand.
  • Be Specific About the Output: Provide details to help the model focus on what's relevant.
  • Use Instructions over Constraints: Tell the model what to do instead of what not to do.
  • Experiment: Try different models, configurations, formats, and word choices to see what works best.
  • Document Your Attempts: Keep a detailed record of your prompts and their results to learn and improve over time.
